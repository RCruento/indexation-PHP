Dans la plupart des cas, les développeurs n'ont pas besoin d'ouvrir une nouvelle instance d'un formulaire, mais plutôt une instance spécifique qui contient déjà des informations. Dans l'article précédent, premier d'une trilogie, j'ai présenté un exemple simple dans lequel ces informations sont représentées par le texte que l'utilisateur charge dans le contrôle étiquette d'une instance de la classe Form. Le code de cette implémentation de base est détaillé dans le listing A. Voyons maintenant comment procéder pour ouvrir une instance existante. Commençons par examiner le modèle général appliqué par C# .Net pour rappeler une instance spécifique d'une classe Form. Rappel d'une instance de formulaire spécifique Imaginons que nous souhaitions utiliser le bouton de l'instance Form2 pour récupérer notre instance de Form1. Avec cette opération, nous nous heurtons aux limites de la création d'une instance d'une classe Form. Pour preuve, ajoutez le code symétrique à l'événement de clic du bouton à l'instance de Form2 pour afficher une instance de Form1: La Figure A illustre la mise en oeuvre de ce code. Lancez maintenant le projet. Saisissez du texte dans la zone de texte de Form1. Cliquez sur le bouton pour le charger dans l'étiquette. Cliquez ensuite sur le bouton pour afficher l'instance de Form2. Sur Form2, cliquez sur le bouton pour afficher une instance de Form1. Vous pouvez répéter cette opération autant de fois que vous le voulez, vous obtiendrez toujours une nouvelle instance de Form1; par conséquent, le texte qui a été chargé par l'utilisateur dans l'étiquette ne sera pas affiché. On trouve généralement bien plus d'informations dans un formulaire Windows qu'un simple morceau de texte, informations saisies par l'utilisateur ou recueillies par un autre moyen. Comment faire alors pour revenir à l'instance en cours de Form1 et conserver ces informations intactes? À l'évidence, nous avons besoin d'une variable à laquelle nous pouvons accéder et qui pointe vers l'instance spécifique de Form1 que nous souhaitons afficher. Une façon d'y parvenir consiste à utiliser une variable publique statique contenant la référence à cette instance. (La différence entre membres de classes statiques et d'instances, comme les variables, tient au fait que les membres statiques n'ont pas besoin d'être instanciés).
